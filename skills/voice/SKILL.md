# Voice - 个人风格与口吻

以你的口吻写作，不是机器的模板。

## 风格来源

**必须**读取 `references/writing-style.json`。

**重要**：样本来源必须是以下之一：
1. 对话记录中用户的发言（由 Agent 从历史对话提取）
2. 用户明确指出是自己编写的文章

**禁止**使用以下作为样本：
- AI 生成的文档（即使内容是用户授权的）
- 从外部来源整理/翻译的内容
- 用户未明确确认的任何内容

## 核心理念

每个人有独特的写作"指纹"——句子节奏、词汇选择、结构偏好、签名用语。这个 skill 捕捉你的风格特征，让输出真正属于你。

## 初始化流程（首次使用）

当 `writing-style.json` 不存在时：

### 选项 A：用户提供样本

```
请提供一些你写过的文字样本（可以是周报、文档、邮件等），
我会分析你的写作风格并存储特征。
```

### 选项 B：从历史对话学习

```
我可以分析我们之前的对话，从中学习你的写作风格。
是否同意我查看历史对话？
```

分析完成后，提取风格特征并写入 `writing-style.json`。

## 风格特征维度

Persona 包含以下维度：

### 1. 句子风格
- **长度偏好**：短句为主 / 均衡 / 长句为主
- **节奏**：紧凑有力 / 舒缓流畅 / 变化丰富
- **结构**：简单句多 / 复合句多 / 混合使用

### 2. 语气特征
- **正式度**：极正式 / 正式 / 半正式 / 口语化
- **情感表达**：克制 / 适度 / 丰富
- **幽默风格**：无 / 自嘲式 / 机智式 / 幽默点缀

### 3. 词汇偏好
- **专业术语**：密集 / 适中 / 稀疏
- **常用词汇**：偏好使用的核心词汇列表
- **避免词汇**：不喜欢的表达方式
- **连接词习惯**：因此 / 所以 / 于是 / 于是乎

### 4. 结构习惯
- **段落长度**：短 / 中 / 长
- **列表偏好**：频繁使用 / 适度使用 / 偶尔使用
- **标题风格**：陈述式 / 提问式 / 简洁式
- **强调方式**：加粗 / 引用 / 代码块 / 无特殊强调

### 5. 签名特征
- **常用短语**：经常出现的表达
- **开头习惯**：常用的开场方式
- **结尾习惯**：常用的收尾方式
- **格式癖好**：表格对齐、特定符号等

## 输出生成规范

### 应用风格的优先级

1. **内容正确性** > 风格匹配度
2. **清晰表达** > 形式美观
3. **用户意图** > 模板习惯

### 生成步骤

1. 理解任务：明确输出目的和受众
2. 确定结构：根据内容类型选择合适框架
3. 应用风格：按 persona 特征调整表达
4. 检查一致：确保全文风格统一
5. 输出呈现：清晰展示最终结果

## 自动增量学习

当用户修改输出时：

```
用户原始输出
    │
    ▼
┌─────────────────────────┐
│ 对比用户修改            │
└────────┬────────────────┘
         │
    ┌────┴────┐
    ▼         ▼
  词汇变化   结构变化
    │         │
    ▼         ▼
 更新词汇   更新结构
 偏好       习惯
    │         │
    └────┬────┘
         │
         ▼
┌─────────────────────────┐
│ 写入 writing-style.json │
└─────────────────────────┘
```

### 学习规则

- **词汇**：用户替换的词，记录偏好
- **结构**：用户重排的段落，调整习惯
- **语气**：用户修改的措辞，更新正式度
- **新增短语**：出现 3 次以上的表达，加入签名特征

### 防止过拟合

- 不因单次修改大幅调整 persona
- 保留核心特征，只微调边缘特征
- 定期询问用户是否需要重置 persona

## 命令

- `/voice` - 以用户风格输出文字
- `/voice --init` - 重新初始化 persona
- `/voice --show` - 显示当前 persona 特征
- `/voice --reset` - 重置 persona

## 示例

### 示例 1：首次使用

```
用户：/voice 帮我写一个周报，总结本周完成的工作

（检测到 writing-style.json 不存在）

AI：首次使用，需要先了解你的写作风格。

你可以：
1. 提供一些你写过的文字样本（周报、文档、邮件等）
2. 让我分析我们的历史对话

你选择哪种方式？
```

### 示例 2：正常使用

```
用户：/voice 写一个技术方案的背景介绍，关于引入新状态管理库

（读取 writing-style.json）

AI：（应用用户风格生成输出）

---

本周我们计划引入 Zustand 作为新的状态管理方案。核心原因有三：

1. **当前问题**：Redux 配置繁琐，新同学上手成本高
2. **技术趋势**：Zustand 体积小、API 简洁，符合"简洁优先"原则
3. **团队能力**：React 18+ 已成为标配，Zustand 与 Concurrent Mode 兼容性好

关键实践：先在非核心模块试点，验证后再推广。
```

### 示例 3：用户修改触发学习

```
AI 输出：...核心原因有三点...

用户修改：...核心原因有三个方面...

（检测到"三点" → "三个方面"）

（更新 vocabulary.preferredWords，记录用户偏好"三个方面"而非"三点"）
```

## 注意事项

1. **隐私**：persona 仅存储写作特征，不存储敏感内容
2. **演进**：风格会随时间变化，定期更新 persona
3. **场景适应**：同一用户不同场景可能有不同风格，当前版本统一处理
4. **边界**：极其正式的公文写作，建议用户确认后再应用
