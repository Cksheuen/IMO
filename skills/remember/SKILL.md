# Remember - 记忆引导技能

**记忆 = 源代码索引 + 关键上下文**

不是代码副本，而是快速定位源代码的路标。

## 核心理念

**抽象层级决定记忆粒度**

| 层级 | 规模 | 记忆内容 | 类比 |
|------|------|----------|------|
| L1 代码片段 | < 50 行，单文件 | 功能描述、关键变量、源路径 | 记住这本书在哪页 |
| L2 模块 | < 200 行，1-5 文件 | 技术栈、架构、接口、源路径 | 记住这章在哪节 |
| L3 系统 | > 200 行，> 5 文件 | 功能概述、核心路径、源目录 | 记住这本书在哪个书架 |

## 目录结构

```
~/.claude/memory/
├── L1-fragments/          # 代码片段级记忆
│   └── {project}_{file}_{name}.md
├── L2-modules/            # 模块级记忆
│   └── {project}_{module}.md
├── L3-systems/            # 系统级记忆
│   └── {project}_system.md
└── index.json             # 全局索引
```

## 层级判断流程

```
代码变更
    │
    ▼
┌─────────────────┐
│ 统计变更范围    │ ← 文件数、行数、跨目录？
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼         ▼
  <50行    50-200行   >200行
  单文件    1-5文件    >5文件
    │         │         │
    ▼         ▼         ▼
   L1        L2        L3
```

## 执行流程

### 记忆写入

```
用户请求"记住 xxx" 或 自动触发
    │
    ▼
┌─────────────────┐
│ 分析代码变更范围 │ ← 文件数、行数统计
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 确定层级 (L1/L2/L3) │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 生成记忆文件    │ ← 只存索引和上下文，不存代码
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 更新索引文件    │ ← 追加到 index.json
└────────┬────────┘
```

### 记忆检索

```
用户请求"回忆 xxx"
    │
    ▼
┌─────────────────┐
│ 搜索索引文件    │ ← 关键词匹配
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
  找到      未找到
    │         │
    ▼         ▼
 返回路径   扩大搜索
 +上下文     或告知无记忆
    │
    ▼
 按需重读源代码
```

## 与其他技能协作

| 技能 | 协作方式 |
|------|----------|
| **eat** | 新知识吸收后，可调用 remember 生成结构化记忆索引 |
| **archive** | 旧记忆归档时，从 index.json 移除条目 |
| **shit** | 精简记忆结构，合并相似记忆 |

## 触发词

- **写入**：`记住`、`记录这个`、`保存记忆`、`更新记忆`
- **检索**：`回忆`、`回忆一下`、`查一下记忆`、`我记得`

## 关键原则

1. **记忆不是代码副本**：只存功能描述和源路径，不存实现代码
2. **层级分明**：不越级记忆，L3 不存接口细节，L2 不存变量
3. **路径可溯**：每个记忆都保留源路径，便于重读
4. **索引优先**：检索先查索引，找到路径后重读源代码
5. **增量更新**：代码变更时更新对应记忆，而非新建

## 为什么不保存代码？

| 问题 | 说明 |
|------|------|
| **内存浪费** | 记忆存储代码与直接读源文件效果相同 |
| **同步问题** | 代码修改后记忆可能落后，导致幻觉 |
| **索引本质** | 记忆的价值在于快速定位，而非内容复制 |

**正确用法**：记忆告诉你"去哪找"，然后重读源代码获取最新实现。
