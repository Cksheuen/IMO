# Remember - 三层记忆管理技能

像人类一样记忆：代码片段记得完整，模块记住技术栈，系统记住宏观但保留检索路径。

## 核心理念

**抽象层级决定记忆深度**

| 层级 | 规模 | 记忆内容 | 类比人类记忆 |
|------|------|----------|--------------|
| L1 代码片段 | < 50 行，单文件 | 完整实现、变量、依赖 | 记住每句话怎么说的 |
| L2 模块 | < 200 行，1-5 文件 | 技术栈、架构、接口 | 记住这章讲了什么方法 |
| L3 系统 | > 200 行，> 5 文件 | 功能概述、核心路径 | 记住这本书在哪个书架 |

## 目录结构

```
~/.claude/memory/
├── L1-fragments/          # 代码片段级记忆
│   └── {project}_{file}_{name}.md
├── L2-modules/            # 模块级记忆
│   └── {project}_{module}.md
├── L3-systems/            # 系统级记忆
│   └── {project}_system.md
└── index.json             # 全局索引
```

## 层级判断流程

```
代码变更
    │
    ▼
┌─────────────────┐
│ 统计变更范围    │ ← 文件数、行数、跨目录？
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼         ▼
  <50行    50-200行   >200行
  单文件    1-5文件    >5文件
    │         │         │
    ▼         ▼         ▼
   L1        L2        L3
```

## 执行流程

### 记忆写入

```
用户请求"记住 xxx"
    │
    ▼
┌─────────────────┐
│ 分析代码变更范围 │ ← 文件数、行数统计
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 确定层级 (L1/L2/L3) │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 生成记忆文件    │ ← 使用对应模板
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 更新索引文件    │ ← 追加到 index.json
└────────┬────────┘
```

### 记忆检索

```
用户请求"回忆 xxx"
    │
    ▼
┌─────────────────┐
│ 搜索索引文件    │ ← 关键词匹配
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
  找到      未找到
    │         │
    ▼         ▼
 返回记忆   搜索源路径
           （重读代码）
```

## 与其他技能协作

| 技能 | 协作方式 |
|------|----------|
| **eat** | 新知识吸收后，可调用 remember 生成结构化记忆 |
| **archive** | 旧记忆归档时，从 index.json 移除条目 |
| **shit** | 精简记忆结构，合并相似记忆 |

## 触发词

- **写入**：`记住`、`记录这个`、`保存记忆`、`更新记忆`
- **检索**：`回忆`、`回忆一下`、`查一下记忆`、`我记得`

## 示例

### 示例1：记录一个工具函数

```
用户：记住这个 debounce 函数

分析：
- 单文件
- < 20 行
- 独立函数

→ 层级: L1
→ 文件: utils_debounce.md
```

### 示例2：记录一个 React 组件模块

```
用户：记住这个表单模块

分析：
- 3 个文件 (Form.tsx, useForm.ts, types.ts)
- ~150 行
- 同一目录

→ 层级: L2
→ 文件: project_form.md
```

### 示例3：记录一个完整系统

```
用户：记住这个权限系统

分析：
- 8 个文件
- 跨 3 个目录
- ~500 行

→ 层级: L3
→ 文件: project_auth-system.md
```

## 关键原则

1. **层级分明**：不越级记忆，L3 不存代码细节
2. **路径可溯**：每个记忆都保留源路径，便于重读
3. **索引优先**：检索先查索引，索引未命中再查文件
4. **增量更新**：代码变更时更新对应记忆，而非新建
