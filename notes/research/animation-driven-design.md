# 动画帧驱动设计原则

> 来源：[Venni Kauppila Thesis - Hollow Knight Animation](https://www.theseus.fi/bitstream/handle/10024/796853/Venni_Kauppila.pdf)、[GDC - Rain World Animation Process](https://www.gdcvault.com/play/1023475/Animation-Bootcamp-Rainworld-Animation)
> 吸收时间：2026-03-28

## 核心洞察

**角色手感 99% 来自精心设计的逐帧动画，代码只负责触发式辅助效果。**

空洞骑士 Team Cherry 在 Photoshop 中手绘每一帧，所有 squash & stretch 在帧内预绘——代码从不用 `scaleX/scaleY` 模拟变形。雨世界的程序化动画同样遵循此原则：物理驱动的是**部件位置**，而非替代手工设计的表现力。

## 触发条件

当设计 2D 游戏角色动画系统时：
- 需要决定某个视觉效果由动画帧还是代码实现
- 角色手感不佳，考虑用代码 tween/粒子补救
- 规划精灵表和动画状态机

## 分工框架

### 动画帧负责（美术控制）

| 效果 | 实现方式 | 原因 |
|------|---------|------|
| Squash & Stretch | 帧内直接画出变形的角色 | 美术对最终效果有完全控制 |
| 移动节奏 | 步伐快慢、身体倾斜角度 | 节奏感来自帧间姿态差异 |
| 攻击打击感 | 快挥(2帧) + 慢收(3帧) 的帧节奏 | "快挥慢收"是打击感核心 |
| 附件跟随 | 斗篷/尾巴的 follow-through 动画 | 延长视觉冲击的持续时间 |
| 角色状态 | 不同状态的独立动画变体 | 状态差异应一眼可辨 |
| 过渡帧 | 1-2 帧的手绘连接动画 | 不是代码插值，是精心设计的姿态 |
| 微动 | idle 的呼吸感、头部微动 | 让角色"活着" |

### 代码负责（程序控制）

| 效果 | 实现方式 | 触发时机 |
|------|---------|---------|
| Hitlag | 暂停游戏 4-6 帧 | 攻击命中帧 |
| 屏幕震动 | Camera shake | 攻击命中、受伤、Boss 攻击 |
| 粒子效果 | Particle emitter | 着地尘土、攻击命中、死亡碎散 |
| 闪烁 | Alpha 切换 (1.0↔0.3) | 无敌帧期间，间隔 120ms |
| 残像 | Afterimage sprite | 快速位移（冲刺/闪现） |
| 音效同步 | Sound.play() | 关键帧触发 |
| 物理击退 | Velocity 设置 | 攻击命中、受伤 |

### 混合区域（程序化附件）

来自雨世界的启发——某些**附属视觉元素**适合程序化：

| 效果 | 适用场景 | 原因 |
|------|---------|------|
| 尾巴/丝带物理 | 角色的装饰性附件 | 需要响应运动方向，逐帧预制成本高 |
| 环境粒子 | 角色周围的氛围粒子 | 数量和方向随状态变化 |
| 眼睛/瞳孔方向 | 注视方向跟随 | 需要实时响应目标位置 |

## 决策框架

```
这个视觉效果应该用什么实现？
    │
    ├─ 角色的姿态/形变/节奏？
    │       → 动画帧（美术画）
    │
    ├─ 瞬时反馈（闪烁/震动/暂停）？
    │       → 代码触发
    │
    ├─ 持续跟随物理的装饰？
    │       → 程序化附件
    │
    └─ 不确定？
            → 先尝试动画帧。
              如果帧数爆炸或需要实时响应 → 改为程序化。
              绝对不要用代码 tween 补救动画不足。
```

## 反模式

| 反模式 | 问题 | 正确做法 |
|--------|------|---------|
| 用 `scaleX/scaleY` 做 squash | 看起来像 bug，不像设计 | 在帧内画出变形后的角色 |
| 用随机 offset 做"不稳定" | 像输入延迟，不像角色特征 | 画不同稳定性等级的动画变体 |
| 用 tween 做攻击拖影 | 缺乏帧间姿态差异的速度感 | 攻击帧本身画出动态模糊线条 |
| 攻击前摇用代码延迟 | 感觉"卡"而非"蓄力" | 画 1 帧蓄力姿态（~16ms） |

## 检查清单

- [ ] 角色的所有 squash & stretch 是否在帧内预绘？
- [ ] 代码是否只在特定帧触发辅助效果（不替代动画）？
- [ ] 附件（尾巴/斗篷）是否有独立的 follow-through？
- [ ] 不同角色状态是否有独立的动画变体（而非代码参数调整）？
- [ ] 过渡帧是否是手绘的（而非代码插值）？
